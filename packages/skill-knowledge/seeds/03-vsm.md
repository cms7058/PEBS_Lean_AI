# 价值流图 VSM

价值流图（Value Stream Mapping）是把产品从订单/原材料到客户交付的**所有信息流与物流**画在一张图上的工具。是精益诊断的"上帝视角"。

## 为什么用 VSM

- 暴露浪费的全貌：把库存、等待、运输、信息延迟一次性看清
- 区分增值时间（VA）与制造周期（Lead Time），通常 VA / Lead Time < 5%
- 形成改善路线图：从当前态（Current State）→ 未来态（Future State）

## 核心数据指标

| 指标 | 含义 | 用途 |
|---|---|---|
| C/T（Cycle Time，循环时间） | 一件产品在该工序的加工时间 | 找瓶颈 |
| C/O（Changeover Time，换型时间） | 切换品种所需时间 | 评估批量经济性 |
| Uptime（设备可用率） | 计划时间内设备可运行的占比 | 评估设备稳定性 |
| Batch Size（批量） | 一次加工/搬运的件数 | 影响 WIP 与流动性 |
| Inventory（库存） | 工序间在制品数量 | 暴露 Mura 与不平衡 |
| Takt Time（节拍） | 客户需求节拍 = 可用工时 / 需求数量 | 设定流动节奏的基准 |
| Lead Time（制造周期） | 物料从入厂到出厂的总时间 | 总体响应速度指标 |
| Value-Added Ratio | VA / Lead Time | 精益化程度的核心比率 |

## 绘图标准符号（精简版）

- 工序框（带数据盒：C/T、C/O、Uptime、人数、班次）
- 库存三角（标注件数与等待天数）
- 物流箭头（推动 = 条纹，拉动 = 圆圈）
- 信息流（细线 = 手工，闪电 = 电子）
- 客户/供应商方框
- 时间线（VA 时间在下，等待时间在上，最终算出 Lead Time 与 Ratio）

## 当前态 → 未来态的常见改善

1. **建立连续流（Continuous Flow）**：合并工序、缩短传递距离、单件流
2. **看板拉动（Pull）**：在无法连续流的位置插入超市与看板
3. **Heijunka 均衡化**：在节拍生产盒中按时间窗口均衡品种
4. **缩短换型（SMED）**：让小批量经济，逼近一个流
5. **稳定关键设备（TPM）**：把 Uptime 拉到 95%+
6. **质量内建（Jidoka / Poka-Yoke）**：让缺陷不流转

## VSM 实施步骤（推荐节奏）

1. 选定**产品族**（Product Family）：用产品-工序矩阵聚类
2. 现场走流程，从客户端逆向走到原材料端，亲自记录数据
3. 画当前态 VSM，算 Lead Time 与 VA Ratio
4. 圈出 7+1 浪费与瓶颈
5. 设计未来态（3–6 个月可达）
6. 制定 Kaizen 改善计划（每个改善爆破对应一项改善）
7. 定期重画，验证改善效果

## 常见误区

- 只画一次贴墙上：VSM 是**活地图**，每 3–6 个月迭代
- 数据靠 ERP 拍脑袋：必须**现场实测**
- 过早追求未来态完美：先抓影响最大的 2–3 项改善
- 忘了信息流：信息延迟常常是瓶颈的真正源头
