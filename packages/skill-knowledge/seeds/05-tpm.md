# TPM 全员生产维护

TPM（Total Productive Maintenance）由日本设备工程学会推广，旨在通过**全员参与的设备管理**实现"零故障、零事故、零缺陷"。

## 核心理念

- 设备问题不只是设备工程师的事，**操作工也是第一责任人**
- 从"事后维修"转向"预防 + 预测"
- 用 OEE 量化设备综合效率

## OEE 公式与三大损失

**OEE = 可用率 × 性能 × 质量**

| 维度 | 公式 | 主要损失 |
|---|---|---|
| 可用率（Availability） | 实际运行时间 / 计划时间 | 故障停机、换型停机 |
| 性能（Performance） | 实际产出 / 理论产出 | 短停、降速运行 |
| 质量（Quality） | 良品数 / 总产出 | 缺陷、返工 |

### 世界级标杆
- **OEE ≥ 85%**：世界级（连续过程行业 ≥ 90%）
- **60–85%**：典型水平
- **< 60%**：有重大改善空间

### 6 大损失（Six Big Losses）

1. 设备故障停机
2. 换型 / 调整时间
3. 短停 / 空转
4. 降速运行
5. 工艺缺陷（启动报废）
6. 产线启动 / 产品切换损失

## TPM 八大支柱

1. **自主保全**：操作工日常清扫、点检、紧固、润滑（4 件事）
2. **计划保全**：维修部门按计划做预防性维修（PM）与预测性维修（PdM）
3. **个别改善**：针对损失大的设备做专项改善（Kobetsu Kaizen）
4. **早期管理**：新设备引入时即纳入 TPM 思维（设计阶段防错）
5. **品质保全**：通过设备稳定性保障产品质量
6. **教育训练**：操作工 + 维修工技能矩阵
7. **事务改善**：办公室 TPM（间接部门效率）
8. **安全环境**：零事故零环境事件

## 自主保全 7 步推进

1. 初期清扫（清扫即点检）
2. 发生源 / 难加工部位对策
3. 制定临时基准书
4. 总点检（操作工掌握设备结构）
5. 自主点检（标准化）
6. 标准化（维护视觉化）
7. 自主管理（持续改善）

## 故障分析：MTBF / MTTR

- **MTBF（Mean Time Between Failures）**：故障间隔均值——越长越稳定
- **MTTR（Mean Time To Repair）**：平均修复时间——越短恢复越快
- 改善方向：MTBF↑（防故障）+ MTTR↓（快修复）

## 与精益其他工具的关系

- **5S** 是 TPM 的基础（看不到设备就管不好设备）
- **OEE** 是 VSM 数据盒的关键字段
- **SMED** 缩短换型 = 提升 OEE 可用率
- **Jidoka 自働化** = 设备自我诊断停机的延伸

## 常见误区

- 把 TPM 当成"操作工兼做维修工"——忽略训练与赋能
- 只抓 PM，不做 Kobetsu Kaizen——长期效果有限
- 忽视早期管理——新设备投产后又重新踩坑
- OEE 数据不准——靠人工记录，掩盖真实损失
