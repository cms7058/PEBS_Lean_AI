# JIT 准时化生产与看板

JIT（Just-In-Time）是丰田生产方式的两大支柱之一（另一支柱是 Jidoka 自働化），核心是**在需要的时间、按需要的数量、提供需要的物品**。看板（Kanban）是实现 JIT 的主要信号系统。

## JIT 三要素

1. **节拍化（Takt Time）**：按客户需求节奏生产
   - Takt = 可用工时 / 客户需求数量
   - 例：8h × 60min ÷ 480 件 = 1 min/件
2. **连续流（Continuous Flow / One-Piece Flow）**：单件流动，最小化 WIP
3. **拉动（Pull）**：由下游需求触发上游补给，而非按预测推动

## 推动 vs 拉动

| 维度 | 推动（Push）| 拉动（Pull） |
|---|---|---|
| 触发方式 | MRP/计划下达 | 下游消耗信号 |
| 库存形态 | WIP 大、过期风险 | WIP 小、近实时补充 |
| 应对变化 | 依赖预测准确 | 自适应实际需求 |
| 异常暴露 | 慢，被库存掩盖 | 快，断流即停 |
| 适用条件 | 长周期、定制化 | 重复性、需求相对稳定 |

## 看板的两大类型

1. **生产看板（Production Kanban）**：授权上游工序生产指定数量
2. **取料看板（Withdrawal Kanban）**：授权下游/物流去上游/超市取货

每张看板上通常包含：物料编号、品名、数量、上游工位、下游工位、容器类型、看板编号 / 总数。

## 看板规则（六条铁律）

1. 后工序按看板从前工序取货
2. 前工序按看板生产，不接到看板不做
3. 没有看板不许搬运、不许生产
4. 看板必须随实物移动
5. 不良品不交给后工序
6. 看板数量逐步减少（持续缩小批量）

## 看板数量计算

```
看板数 N = (D × L × (1 + α)) / C
```
- D：单位时间需求量
- L：补给周期（生产 + 等待 + 搬运）
- α：安全系数（通常 0.1–0.3）
- C：每个容器的件数

例：D = 100 件/小时，L = 1 小时，α = 0.2，C = 20 件 → N = (100×1×1.2)/20 = 6 张看板

## 超市（Supermarket）

当无法做到完全连续流时，用"超市"作为拉动缓冲：

- 上游工序生产到超市
- 下游工序按看板从超市取
- 超市最大库存有上限（看板总数控制）
- 适用于：节拍不匹配、共用设备、远距离工序

## 电子看板与可视看板

- **物理看板**：卡片、磁吸、灯柱（最直观）
- **电子看板（e-Kanban）**：MES/WMS 集成，适合多工厂、跨地域
- 注意：电子看板易"看不见"，配合大屏、安灯、邮件提醒

## JIT 推进的前置条件

- 设备稳定（TPM 支撑可用率）
- 质量稳定（Jidoka + Poka-Yoke 防止次品流转）
- 换型快速（SMED 让小批量可行）
- 供应商协同（外部供应链能否准时供货）
- 计划均衡（Heijunka 防止 Mura）

## 常见误区

- 一上来就全厂拉动——前置条件不具备会导致频繁断料
- 看板数量定死不动——精益要求逐步减少
- 用 ERP/MRP 推完一段又用看板拉一段——混合系统易混乱
- 把"零库存"当目标——零库存是手段不是目标
