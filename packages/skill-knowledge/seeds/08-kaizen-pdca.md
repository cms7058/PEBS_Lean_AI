# Kaizen 改善与 PDCA 循环

Kaizen（改善 / 改善）是日语"持续小步改进"的意思。与"创新（Innovation）"的大跨步不同，Kaizen 强调**全员参与、每天一小步、靠累积出大变化**。

## Kaizen 的两层含义

1. **全员改善（Daily Kaizen）**：一线员工每天 5–15 分钟的小改善
2. **改善活动（Kaizen Event / Blitz）**：跨部门 3–5 天的专项改善周

两者互补：日常改善培养土壤，专项改善突破瓶颈。

## PDCA 循环（戴明环）

PDCA 是改善的标准节奏：

| 阶段 | 内容 | 关键产物 |
|---|---|---|
| **P（Plan 计划）** | 现状把握、目标设定、原因分析、对策制定 | 改善计划书 |
| **D（Do 执行）** | 小范围试点 | 试点数据 |
| **C（Check 验证）** | 与目标对比、效果确认 | 验证报告 |
| **A（Act 标准化）** | 标准化、横向展开 / 调整方案重启 PDCA | 新标准书 |

### PDCA 与 SDCA 配对

- **SDCA**（Standardize-Do-Check-Act）：维持标准的循环
- **PDCA**：突破现状的循环
- 两者交替：S→S→S→P（突破）→S→S→S→P（再突破）

## 5 Why 根因追问

Kaizen 与 PDCA 的灵魂在 P 阶段的根因分析。"5 Why"是最朴素也最有力的工具：

```
现象：传送带停了
1 Why? → 保险丝烧了
2 Why? → 轴承负荷过大
3 Why? → 轴承润滑不足
4 Why? → 润滑泵未抽到油
5 Why? → 泵的轴磨损未检查
对策：列入定期点检
```

注意：5 不是固定数，问到能"采取行动的根因"为止。

## 鱼骨图（Ishikawa / Cause-Effect Diagram）

把可能原因按 5M1E 分类：

- **人（Man）**：技能、经验、士气、班次
- **机（Machine）**：设备、工装、模具、精度
- **料（Material）**：来料质量、规格、批次差异
- **法（Method）**：作业标准、流程、参数
- **环（Environment）**：温湿度、照明、振动、清洁
- **测（Measurement）**：测量方法、量具、人为判读

每根分支再追问 5 Why 找到末梢根因。

## A3 报告

丰田的标准改善文档，一张 A3 纸完成：

1. 背景 / 现状
2. 目标
3. 当前状态分析（含数据、5 Why、鱼骨）
4. 根因
5. 对策方案
6. 实施计划（5W2H）
7. 验证结果
8. 标准化与横展

## Kaizen 文化建设

- **改善提案制度**：每月人均 1+ 条（丰田为 10+ 条）
- **可视化看板**：改善数量、节省金额、完成率
- **管理层走动（Gemba Walk）**：领导每周到现场看
- **激励机制**：精神为主（表彰、晋升）+ 物质适度
- **失败包容**：试错文化，A 阶段允许"调整方案重启"

## 改善活动（Kaizen Event）周

- **Day 1**：现场观察 + 数据收集
- **Day 2**：根因分析 + 方案设计
- **Day 3**：动手改造（移设备、画线、做防错）
- **Day 4**：试运行 + 调整
- **Day 5**：标准化 + 成果发表

## 与其他工具的关系

- VSM 找出改善靶点
- 5S / SMED / TPM 是 Kaizen 的常见主题
- 8D / DMAIC 是 PDCA 的扩展版本（针对客户问题 / 黑带项目）
- A3 是 Kaizen 的标准沟通载体

## 常见误区

- 只有大项目才叫改善——丢失全员土壤
- A 阶段缺位——改了就忘，无法横展
- 没有标准就改善——破坏稳定基础
- 用 KPI 压指标——伪改善、数据造假
