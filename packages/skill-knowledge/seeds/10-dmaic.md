# DMAIC 六西格玛改善框架

DMAIC 是六西格玛（Six Sigma）项目的标准 5 阶段方法论：**Define（定义）→ Measure（测量）→ Analyze（分析）→ Improve（改善）→ Control（控制）**。

适用场景：**慢性问题、变异主导、需要数据驱动的中长期项目**。

## D — Define（定义阶段）

确定项目的"是什么、为什么、做多少"。

- **项目章程（Project Charter）**：背景、问题陈述、目标、范围、团队、时间表、ROI 估算
- **VOC（Voice of Customer）**：客户对该产品/服务的关键要求
- **CTQ（Critical-to-Quality）**：可测量的关键质量特性
- **SIPOC 图**：Supplier-Input-Process-Output-Customer 高阶流程图
- 输出：批准的项目章程

## M — Measure（测量阶段）

把现状量化。

- **过程图（Process Map）**：详细工序流程
- **测量系统分析（MSA / Gage R&R）**：确认测量本身可信
- **数据收集计划**：测量方法、频率、样本量、记录格式
- **基线性能（Baseline）**：当前缺陷率、Cpk、过程能力
- 输出：基线数据 + MSA 报告

### 关键能力指数

- **Cp / Cpk**：短期过程能力（≥ 1.33 良好，≥ 1.67 卓越）
- **Pp / Ppk**：长期过程能力
- **DPMO**：每百万机会缺陷数（六西格玛 = 3.4 DPMO）
- **σ Level**：过程性能等级（3σ ≈ 66,800 DPMO；6σ ≈ 3.4 DPMO）

## A — Analyze（分析阶段）

找根因。

- **数据分析**：直方图、箱型图、Pareto 图（80/20）、控制图、散点图
- **假设检验**：t 检验、ANOVA、卡方检验
- **回归分析**：找 X 与 Y 的定量关系
- **失效模式分析（FMEA）**：风险优先级 RPN = 严重度 × 发生频率 × 探测难度
- **5 Why / 鱼骨图**：定性分析
- 输出：经数据验证的关键 X（影响因素）

## I — Improve（改善阶段）

设计并验证对策。

- **DOE（Design of Experiments，实验设计）**：用最少试验找最优参数
  - 全因子 / 部分因子 / 响应面法（RSM）
- **方案选择**：与基线对比、成本-收益、风险评估
- **试点（Pilot）**：小批量验证
- **风险识别**：FMEA 评估对策本身
- 输出：经验证的改善方案 + 试点数据

## C — Control（控制阶段）

防止反弹。

- **控制计划（Control Plan）**：每个关键 X 的监控方法、频率、规格、超规处置
- **SPC 控制图**：实时监控（X-bar R / I-MR / p / np / c / u 图）
- **标准作业书更新**
- **培训与移交**：把项目交回过程负责人
- **验证持续效果**：3–6 个月后复测
- 输出：控制计划 + 标准化文档 + 项目结案报告

## DMAIC 与精益的融合（Lean Six Sigma）

- DMAIC 提供**结构化项目框架**
- 精益工具填充**改善手段**：VSM、5S、SMED、Kaizen Event、Pull
- 节奏：D-M 用精益工具快速暴露浪费 → A 用统计方法确认 → I 落实精益对策 → C 用 SPC 巩固

## DMAIC 角色（带状腰带）

| 角色 | 培训 | 职责 |
|---|---|---|
| **黄带（YB）** | 1–2 周 | 项目成员，掌握基础工具 |
| **绿带（GB）** | 2–4 周 | 兼职项目负责人，年完成 1–2 个项目 |
| **黑带（BB）** | 4–5 周 | 全职项目负责人，年完成 4–6 个项目 |
| **黑带大师（MBB）** | 8 周+ | 培训、辅导、项目筛选 |
| **倡导者（Champion）** | 1–2 天 | 高管，推动战略与资源 |

## DMAIC 与 8D 的对比

| 维度 | 8D | DMAIC |
|---|---|---|
| 触发 | 客户投诉、急性问题 | 慢性变异、战略改善 |
| 周期 | 数天–数周 | 3–6 月 |
| 数据强度 | 中 | 高（统计学为骨架） |
| 团队规模 | 5–8 人 | 5–10 人 + Champion |
| 输出深度 | 修复 + 横展 | 系统化能力提升 |

## 常见误区

- D 阶段问题陈述不清——后续都偏离方向
- M 阶段跳过 MSA——数据不准白干
- A 阶段不做假设检验——靠经验跳到 I
- C 阶段无人接管——3 个月后反弹

## 项目筛选标准

- 财务收益 ≥ 50–100 万 / 年
- 周期 ≤ 6 个月
- 数据可获取
- 与战略 KPI 对齐
- Champion 支持
